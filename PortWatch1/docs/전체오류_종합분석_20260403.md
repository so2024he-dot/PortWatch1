## 1. INI 파일 분석 (오류코드 수정할 부분 및 생성코드.ini)

### 1-1. Maven Build (14:26:29) - 성공

→ Maven Build는 성공. 단, Spring Bean 설정 오류는 런타임(Tomcat 기동 시)에만 발견됨.

### 1-2. STS Console (18:22:17) - keepaliveTime BeanCreationException
```
WARN: Exception encountered during context initialization - cancelling refresh attempt:
org.springframework.beans.factory.BeanCreationException:
  Error creating bean with name 'dataSource' defined in [/WEB-INF/spring/root-context.xml]:
  Error setting property values;
  nested exception is org.springframework.beans.NotWritablePropertyException:
    Invalid property 'keepaliveTime' of bean class [com.zaxxer.hikari.HikariDataSource]:
    Bean property 'keepaliveTime' is not writable or has an invalid setter method.
```

**원인 분석**:
```
HikariCP 3.4.5에서 keepaliveTime 속성:
  - Java 코드로 직접 접근: HikariConfig.setKeepaliveTime() ← 존재함
  - Spring XML bean property 주입: <property name="keepaliveTime"/> ← 불가!

Spring XML 방식은 BeanWrapper를 통해 setter를 찾는데,
HikariCP 3.4.5의 setKeepaliveTime()은 Spring의 BeanWrapper가 인식하지 못하는 형태.
```

**수정 내용** (이번 세션에서 적용 완료):
```xml
<!-- 제거된 코드 (root-context.xml line 119-123) -->
<!--
<property name="keepaliveTime" value="60000"/>
-->

<!-- 현재 상태 (정상) -->
<property name="connectionTestQuery" value="SELECT 1"/>
<!-- ✅ -1: 기동 시 DB 연결 실패해도 Tomcat 기동 허용 -->
<property name="initializationFailTimeout" value="-1"/>
```

**대안 (향후 필요 시)**: keepaliveTime과 동일한 효과는 이미 구현됨:
```xml
<property name="connectionTestQuery" value="SELECT 1"/>  <!-- 사용 전 검증 -->
<property name="maxLifetime" value="180000"/>              <!-- 3분마다 연결 교체 -->
<property name="idleTimeout" value="120000"/>              <!-- 2분 유휴 후 교체 -->
```

### 1-3. STS Console - 추가 오류

심각: 이전 오류들로 인해 컨텍스트 []의 시작이 실패했습니다.

## 2. EC2 스크린샷 분석 (144131 - HikariPool Timeout)

### 2-1. db-check 결과
```json
{
  "dbConnected": false,
  "error": "MyBatisSystemException",
  "message": "HikariPool-1 - Connection is not available, request timed out after 30000ms",
  "diagnostics": "[HikariCP 30초 타임아웃] ..."
}
```

### 2-2. 원인 (구버전 WAR 배포)
```
타임라인:
  April 2 ~18:22 → keepaliveTime 추가된 root-context.xml
  April 3  14:26 → Maven install (keepaliveTime 포함된 WAR 빌드)
  April 3  14:41 → EC2에 14:26 WAR 배포
  April 3  14:41 → Spring Context 실패 OR HikariCP 연결 실패

문제: 14:26 WAR에 포함된 root-context.xml이 이전 버전 (혹은 jdbcUrl 문제)
```

### 2-3. MySQL CLI vs JDBC 차이
```
mysql CLI 연결 성공:
  → OS 수준 TCP 연결 + mysql_native_password 인증
  → 모든 auth 플러그인 자동 협상

JDBC(HikariCP) 연결 실패 가능 원인:
  → MySQL 8.4.7 기본 auth: caching_sha2_password
  → JDBC URL에 allowPublicKeyRetrieval=true 없으면 PublicKey 오류
  → 현재 root-context.xml에는 포함됨 (과거 WAR에는 없었을 수 있음)
```

### 2-4. 해결책 — 새 WAR 빌드 & 배포
현재 root-context.xml (keepaliveTime 제거 후) 기준으로 Maven install → EC2 배포.

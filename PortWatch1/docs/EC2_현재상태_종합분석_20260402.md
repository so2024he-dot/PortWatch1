## 2. PuTTy 화면 EC2 명령 분석

```
cat /opt/tomcat/bin/setenv.sh → Permission denied  ← 이전 setenv.sh 600 권한 문제
sudo tee /opt/tomcat/bin/setenv.sh << 'SETENV'    ← 올바른 방식으로 재생성
> #!/bin/bash
> export JAVA_OPTS="-Djava.net.preferIPv4Stack=true -Xms256m -Xmx512m"
> SETENV
sudo chmod 755 /opt/tomcat/bin/setenv.sh           ← 권한 정상화
sudo systemctl stop tomcat                          ← Tomcat 정지
sudo cp /home/ubuntu/portwatch.war /opt/tomcat/webapps/ROOT.war  ← 새 WAR 배포
sudo rm -rf /opt/tomcat/webapps/ROOT/              ← 기존 폴더 삭제
sudo systemctl start tomcat                         ← Tomcat 시작
ps aux | grep java | grep -o 'preferIPv4Stack=true' → preferIPv4Stack=true ✅
sleep 40 && curl -s http://localhost:8080/debug/db-check → dbConnected: false ❌
```

### ✅ 확인된 정상 항목
- `preferIPv4Stack=true` → **IPv4 강제 설정 정상 작동**
- setenv.sh chmod 755 → Permission denied 해결됨
- 새 WAR 배포 완료 (17:11:29 빌드본)

### ❌ 아직 미해결: DB 연결 실패
- HikariPool-1 - Connection is not available, request timed out after 30000ms
- IPv4는 적용됐는데도 DB 연결 실패 → 추가 진단 필요

## 3. DB 연결 실패 원인 3단계 진단 (지금 PuTTy에서 실행)

```bash
# ─────────────────────────────────────────────
# STEP 1: IPv4 확인 (이미 성공했지만 재확인)
# ─────────────────────────────────────────────
ps aux | grep java | grep -o 'preferIPv4Stack=true'
# → "preferIPv4Stack=true" 출력돼야 함

# ─────────────────────────────────────────────
# STEP 2: RDS까지 TCP 3306 연결 직접 테스트
# ─────────────────────────────────────────────
nc -zv portwatch-db-seoul.cpggwqm1goo3.ap-northeast-2.rds.amazonaws.com 3306
# → "Connection to ... 3306 port [tcp/mysql] succeeded!" 가 나와야 함
# → 실패하면 보안그룹 문제

# ─────────────────────────────────────────────
# STEP 3: MySQL 직접 연결 (Java 없이 순수 TCP)
# ─────────────────────────────────────────────
mysql -h portwatch-db-seoul.cpggwqm1goo3.ap-northeast-2.rds.amazonaws.com \
      -P 3306 -u admin -p'portwatch2026!!abcd' portwatch -e "SELECT 1"
# → "1" 출력되면 DB 정상, Java 설정 문제
# → 실패하면 네트워크/인증 문제

# ─────────────────────────────────────────────
# STEP 4: 배포된 WAR의 jdbcUrl 확인
# ─────────────────────────────────────────────
unzip -p /opt/tomcat/webapps/ROOT.war WEB-INF/spring/root-context.xml \
  | grep -o 'jdbcUrl.*"'
# → cpggwqm1goo3 (숫자 1) 이어야 함 (i 아님)
# → 한 줄로 작성되어 있어야 함

# ─────────────────────────────────────────────
# STEP 5: Tomcat 시작 로그 오류 확인
# ─────────────────────────────────────────────
sudo grep -E "(HikariPool|ERROR|WARN|Communications)" \
  /opt/tomcat/logs/catalina.out | tail -30
```

## 4. STS wtpwebapps 오류 완전 해결법 (5번째 발생, 17:14:10)

### 오류 메시지
```
wtpwebapps\portwatch1 is not a directory or war file
```

### 원인
Maven Clean → `target/` 전체 삭제 → STS WTP가 참조하는 `wtpwebapps/portwatch1/` 폴더 소멸

### 해결 순서 (반드시 이 순서로!)

```
① STS 상단 메뉴: Project → Clean... → 체크박스 "All Projects" → OK
   (잠시 빌드가 진행됨 — 완료까지 기다림)

② Servers 탭에서 Tomcat 서버 선택
   → 우클릭 → "Add and Remove..."
   → portwatch1이 "Configured" 쪽에 있는지 확인
   → 만약 없으면 "Available" → "Add" → Finish

③ Servers 탭에서 Tomcat 서버 선택
   → 우클릭 → "Clean..."  (wtpwebapps 폴더 재생성)
   → OK

④ Servers 탭에서 Tomcat 서버 선택
   → 우클릭 → "Publish"  (파일 복사)

⑤ Servers 탭에서 Tomcat 서버 선택
   → Start (▶ 버튼)
```

### 영구 해결책 (같은 오류 재발 방지)
```
Servers 탭 → Tomcat 더블클릭
→ "Server Locations" 섹션
→ "Use Tomcat installation (takes control of Tomcat installation)" 선택
→ Deploy path: wtpwebapps (기본값 유지)
→ 저장 (Ctrl+S)
→ 이후 Maven Clean을 해도 wtpwebapps 경로가 유지됨
```

## 5. 로그인 실패 원인 및 해결

### 상황
- `test@test.com` → 회원가입 시 "이미 사용 중인 이메일" → DB에 test@test.com 존재 ✅
- `test@test.com` 로그인 → "이메일 또는 비밀번호가 올바르지 않습니다" → 비밀번호 불일치 ❌

### 원인
DB에 test@test.com 계정이 있지만 저장된 BCrypt 해시가 현재 입력 비밀번호와 불일치.

### 해결: MySQL에서 비밀번호 초기화

```sql
-- EC2 PuTTy에서 MySQL 접속
mysql -h portwatch-db-seoul.cpggwqm1goo3.ap-northeast-2.rds.amazonaws.com \
      -P 3306 -u admin -p

-- DB 선택
USE portwatch;

-- 1. test@test.com 계정 확인
SELECT member_id, member_email, member_name, member_status
  FROM MEMBER
 WHERE member_email = 'test@test.com';

-- 2. 비밀번호를 'portwatch123'으로 초기화
--    (아래 BCrypt 해시는 'portwatch123'에 대응)
UPDATE MEMBER
   SET member_pass = '$2a$10$0GBSb04xfIUriIM/5WwqhSuKgZHeltHn.4qFqnLIlojkISte/pSgSC'
 WHERE member_email = 'test@test.com';

-- 3. 상태 확인 (ACTIVE여야 로그인 가능)
UPDATE MEMBER SET member_status = 'ACTIVE'
 WHERE member_email = 'test@test.com';

-- 4. 전체 회원 확인
SELECT member_id, member_email, member_status FROM MEMBER;
```

**비밀번호: `portwatch123` (위 BCrypt 해시에 해당)**

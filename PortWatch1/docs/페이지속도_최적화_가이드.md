## 1. 현재 속도 문제 원인 분석

### 1-1. AWS RDS 연결 지연 (최대 30초 블로킹)

**문제 코드** (`root-context.xml`):
```xml
<!-- ❌ 현재: initializationFailTimeout=30000 → Tomcat 기동 시 30초 대기 -->
<property name="initializationFailTimeout" value="30000"/>
<property name="connectionTimeout" value="30000"/>
```

**증상:**
- 첫 번째 페이지 요청 시 HikariCP가 DB 연결을 초기화하며 최대 30초 대기
- 연결 실패 시 `CommunicationsException` → 전 페이지가 에러 표시

**해결책:**
```xml
<!-- ✅ 수정: 빠른 연결 실패 감지 + 연결 재시도 -->
<property name="initializationFailTimeout" value="-1"/>  <!-- 기동 실패 방지 -->
<property name="connectionTimeout" value="5000"/>         <!-- 5초로 단축 -->
<property name="minimumIdle" value="2"/>                  <!-- 유휴 연결 유지 -->
<property name="maximumPoolSize" value="10"/>             <!-- 풀 크기 증가 -->
```

### 1-2. 뉴스/주식 목록 - 페이지네이션 없이 전체 조회

**문제 코드** (`NewsMapper.xml`):
```xml
<!-- ❌ LIMIT 없이 전체 조회 → 데이터 많을수록 느려짐 -->
<select id="selectAllNews" resultMap="NewsResultMap">
    SELECT news_id, title, stock_code, news_url, published_date, name
    FROM news ORDER BY published_date DESC
</select>
```

**문제 코드** (`StockMapper.xml`):
```xml
<!-- ❌ 전체 주식 조회 → STOCK 테이블 200건 이상 시 느려짐 -->
<select id="findAll" resultMap="StockResultMap">
    SELECT stock_code, stock_name, ... FROM STOCK ORDER BY country ASC, stock_code ASC
</select>
```

**해결책:**
```xml
<!-- ✅ 페이지네이션 추가 -->
<select id="selectNewsPage" resultMap="NewsResultMap">
    SELECT news_id, title, stock_code, news_url, published_date, name
    FROM news ORDER BY published_date DESC
    LIMIT #{pageSize} OFFSET #{offset}
</select>

<select id="findAllPaged" resultMap="StockResultMap">
    SELECT stock_code, stock_name, country, market, market_type, industry,
           current_price, price_change, change_rate, volume, market_cap,
           updated_at, created_at
    FROM STOCK ORDER BY country ASC, stock_code ASC
    LIMIT #{pageSize} OFFSET #{offset}
</select>
```

### 1-3. 뉴스 크롤링이 HTTP 요청과 동일 스레드에서 실행

**문제 코드** (`NewsController.java`):
```java
// ❌ POST /news/refresh → 크롤링이 완료될 때까지 HTTP 응답 블로킹
@PostMapping("/refresh")
@ResponseBody
public ResponseEntity<Map<String, Object>> refreshNews() {
    int count = newsService.crawlAndSaveNews();  // 최대 10초 소요
    ...
}
```

**NaverNewsCrawler.java**:
```java
// ❌ Jsoup.connect() 동기 I/O → 10초 타임아웃 동안 스레드 점유
Document doc = Jsoup.connect(NAVER_FINANCE_NEWS_URL)
        .timeout(TIMEOUT)  // TIMEOUT = 10000ms
        .get();
```

**해결책 - 비동기 처리:**
```java
// ✅ @Async로 백그라운드 실행
@PostMapping("/refresh")
@ResponseBody
public ResponseEntity<Map<String, Object>> refreshNews() {
    newsService.crawlAndSaveNewsAsync();  // 즉시 리턴, 백그라운드 크롤링
    Map<String, Object> response = new HashMap<>();
    response.put("success", true);
    response.put("message", "크롤링이 시작되었습니다.");
    return ResponseEntity.ok(response);
}
```

### 1-4. 스케줄러 충돌 - 다중 스케줄러 동시 실행

| 스케줄러 | 실행 주기 | DB 사용 |

| `NewsScheduler.crawlKoreanNewsAutomatically()` | 30분마다 | ✅ 사용 |
| `NewsScheduler.crawlUSNewsAutomatically()` | 1시간마다 | ❌ TODO만 있음 |
| `StockPriceScheduler` | (별도) | ✅ 사용 |
| `StockUpdateScheduler` | (별도) | ✅ 사용 |

## 2. 즉시 적용 가능한 최적화 (우선순위 순)

| 우선순위 | 항목 | 예상 개선 | 난이도 |
|---|---|---|---|
| 🔴 1순위 | **JDBC URL 오타 수정** (DB 연결 복구) | 전체 기능 복구 | ⭐ 쉬움 |
| 🔴 2순위 | `connectionTimeout` 30초→5초 단축 | -25초 | ⭐ 쉬움 |
| 🟡 3순위 | 뉴스 목록 `LIMIT 50` 제한 | -1~2초 | ⭐⭐ 보통 |
| 🟡 4순위 | 주식 목록 페이지네이션 (50건/페이지) | -2~3초 | ⭐⭐ 보통 |
| 🟢 5순위 | CDN → 로컬 리소스 전환 | -0.5~2초 | ⭐⭐ 보통 |
| 🟢 6순위 | 뉴스 크롤링 비동기 처리 | -10초 (블로킹 제거) | ⭐⭐⭐ 어려움 |

## 3. AWS EC2 Tomcat JVM 설정 최적화

**현재 설정** (`/etc/systemd/system/tomcat.service`):
```
Environment="JAVA_OPTS=-Djava.net.preferIPv4Stack=true -Djava.security.egd=file:/dev/./urandom"
Environment="CATALINA_OPTS=-Xms512M -Xmx1024M"
```

**권장 설정:**
```
Environment="JAVA_OPTS=-Djava.net.preferIPv4Stack=true \
  -Djava.security.egd=file:/dev/./urandom \
  -Duser.timezone=Asia/Seoul"
Environment="CATALINA_OPTS=-Xms256M -Xmx768M \
  -XX:+UseG1GC \
  -XX:MaxGCPauseMillis=200 \
  -XX:+HeapDumpOnOutOfMemoryError"
```

> **주의:** `-Xmx1024M`은 t3.micro(1GB RAM) 인스턴스에서 OOM 위험. `-Xmx768M`으로 낮춤

## 5. 뉴스 자동 새로고침 (Polling 방식)

현재 `news/list.jsp`에는 자동 새로고침이 없음.
아래 JavaScript를 `news/list.jsp`에 추가하면 5분마다 새 뉴스 확인:

```javascript
// ✅ 5분마다 새 뉴스 확인 (폴링)
let lastNewsId = 0;
setInterval(function() {
    fetch('/api/news/recent?limit=5&since=' + lastNewsId)
        .then(res => res.json())
        .then(data => {
            if (data.newsList && data.newsList.length > 0) {
                showNewNewsAlert(data.newsList.length);
                lastNewsId = data.newsList[0].newsId;
            }
        })
        .catch(err => console.warn('뉴스 확인 실패:', err));
}, 300000); // 5분 = 300,000ms

function showNewNewsAlert(count) {
    const alert = document.createElement('div');
    alert.className = 'alert alert-info';
    alert.innerHTML = '📰 새 뉴스 ' + count + '건이 있습니다. <a href="#" onclick="location.reload()">새로고침</a>';
    document.querySelector('.container').prepend(alert);
}
```

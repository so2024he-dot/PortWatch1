## 1. setenv.sh Permission denied 분석

### 1-1. 발생 현상
```
cat /opt/tomcat/bin/setenv.sh
cat: /opt/tomcat/bin/setenv.sh: Permission denied
```

### 1-2. 원인
- `/opt/tomcat/bin/` 디렉토리는 tomcat 사용자 소유
- ubuntu 계정에 read 권한 없음 (`chmod 700` 또는 `chmod 744` 상태)

### 1-3. 해결 (기능적 문제 아님)
```bash
sudo chmod 644 /opt/tomcat/bin/setenv.sh
cat /opt/tomcat/bin/setenv.sh
```

**예상 출력**:
```bash
#!/bin/bash
export JAVA_OPTS="-Djava.net.preferIPv4Stack=true -Xms256m -Xmx512m"
```

### 1-4. preferIPv4Stack 적용 확인
`Permission denied`가 뜨더라도 실제 JVM 옵션은 적용되어 있음:
```bash
ps aux | grep tomcat | grep preferIPv4
```
→ `-Djava.net.preferIPv4Stack=true` 가 JVM args에 있으면 정상

## 2. HikariPool-1 연결 타임아웃 분석

### 2-1. 오류 메시지
```json
{
  "dbConnected": false,
  "error": "MyBatisSystemException",
  "message": "HikariPool-1 - Connection is not available, request timed out after 30000ms"
}
```

### 2-2. 근본 원인 진단

**원인 A: 구버전 WAR 배포 (최우선 원인)**
```
스크린샷에서 확인된 mapper 경로:
  오류: mapper/MemberMapper_pkg.xml
  현재: mappers/MemberMapper_mapper_pkg.xml
→ 14:26 이전 빌드본이 EC2에 배포되어 있음
```

**원인 B: jdbcUrl 파라미터 문제 (수정 완료)**
- `keepaliveTime` → Spring bean property 없음 → BeanCreationException
- `autoReconnect=true` → MySQL 8.x deprecated → 제거 완료
- `useUnicode=true` 누락 → 추가 완료
- `tcpKeepAlive=true` 누락 → 추가 완료

**원인 C: caching_sha2_password 인증**
- MySQL 8.4.7 기본 인증 방식
- `allowPublicKeyRetrieval=true` 필수 → 현재 포함됨 ✅

### 2-3. 해결 절차 (순서대로 실행)
```bash
# Step 1: STS Maven install (로컬)
# Project → Run As → Maven install → BUILD SUCCESS

# Step 2: WinSCP 전송
# 로컬: C:\Users\소현석\git\PortWatch1\PortWatch1\target\portwatch-1.0.0-BUILD-SNAPSHOT.war
# 원격: /home/ubuntu/portwatch.war

# Step 3: EC2 배포
sudo systemctl stop tomcat
sudo cp /home/ubuntu/portwatch.war /opt/tomcat/webapps/ROOT.war
sudo rm -rf /opt/tomcat/webapps/ROOT/
sudo systemctl start tomcat
sleep 45 && curl -s http://localhost:8080/debug/db-check | python3 -m json.tool
```

**성공 응답**:
```json
{
  "dbConnected": true,
  "memberFound": true
}
```

## 3. HikariPool 연결 실패 시 Tomcat 로그 진단

### 3-1. 실시간 로그 확인
```bash
sudo tail -f /opt/tomcat/logs/catalina.out
```

### 3-2. 핵심 오류 패턴 검색
```bash
# HikariCP 관련 오류
sudo grep -n "HikariPool\|HikariCP\|hikari" /opt/tomcat/logs/catalina.out | tail -30

# JDBC 연결 오류
sudo grep -n "Communications link failure\|Connection refused\|timed out" /opt/tomcat/logs/catalina.out | tail -20

# BeanCreationException (Spring context 실패)
sudo grep -n "BeanCreationException\|NotWritablePropertyException" /opt/tomcat/logs/catalina.out | tail -10

# MySQL 인증 오류
sudo grep -n "caching_sha2_password\|Access denied\|Public Key" /opt/tomcat/logs/catalina.out | tail -10
```

### 3-3. 오류 유형별 조치

| 오류 메시지 | 원인 | 조치 |
|------------|------|------|
| `NotWritablePropertyException: keepaliveTime` | HikariCP 3.4.5 미지원 속성 | root-context.xml에서 제거 |
| `Communications link failure` | jdbcUrl 개행 포함 or 호스트 오타 | jdbcUrl 한 줄 확인, 호스트명 확인 |
| `Connection timed out` | 보안그룹 차단 or 호스트 오타 | RDS 보안그룹 3306 허용 확인 |
| `Public Key Retrieval is not allowed` | MySQL 8.x 인증 문제 | `allowPublicKeyRetrieval=true` 추가 |
| `Access denied for user 'admin'` | 비밀번호 틀림 or 권한 없음 | MySQL에서 권한 확인 |

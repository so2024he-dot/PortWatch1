## 2. 레이어별 연계 관계도

```
[Browser] → [Controller] → [Service] → [Mapper Interface] → [XML] → [RDS MySQL]

로그인 흐름:
POST /member/login
  └→ MemberController.login()
       └→ MemberServiceImpl.login()
            └→ MemberMapper.findByEmail()          ← mapper 패키지
                 └→ MemberMapper_mapper_pkg.xml    ← SELECT * FROM MEMBER
                      └→ MEMBER 테이블 (RDS)

뉴스 조회 흐름:
GET /news/list
  └→ NewsController.newsList()
       └→ NewsServiceImpl.getRecentNews(50)
            └→ NewsDAO.selectRecentNews(limit)     ← persistence 패키지
                 └→ NewsMapper.xml                 ← SELECT FROM news LIMIT
                      └→ news 테이블 (RDS)

주식 목록 흐름:
GET /stock/list
  └→ StockController.list()
       └→ StockServiceImpl.getAllStocks()
            └→ StockMapper.findAll()               ← mapper 패키지
                 └→ StockMapper.xml                ← SELECT FROM STOCK
                      └→ STOCK 테이블 (RDS)

크롤링 흐름:
[NewsScheduler - 30분마다]
  └→ NewsServiceImpl.crawlAndSaveNews()
       └→ NaverNewsCrawler.crawlNews()             ← Jsoup으로 네이버 금융 크롤링
            └→ NewsDAO.insertNews()                ← DB 저장
                 └→ news 테이블 (RDS)
```

**수정 방법 (PuTTy EC2):**
```bash
# nano로 직접 편집 (sed 사용 금지!)
sudo nano /opt/tomcat/webapps/ROOT/WEB-INF/spring/root-context.xml
# jdbcUrl 또는 url 속성값을 올바른 endpoint로 수정

sudo systemctl restart tomcat
sleep 20
curl -s http://localhost:8080/debug/db-check
```

### Bug #7: MemberMapper_mapper_pkg.xml - member_email UNIQUE 처리 미흡

```xml
<!-- member_email은 UNIQUE이지만 ACTIVE 상태 필터로 조회 -->
<select id="findByEmail">
    SELECT * FROM MEMBER WHERE member_email = #{memberEmail} AND member_status = 'ACTIVE'
</select>
```

**문제:** member_email이 DB에 있지만 INACTIVE 상태면:
- `checkEmailAvailable()` → null 반환 → `available=true` → 중복 INSERT 허용
- INSERT 시 UNIQUE 제약 위반 → 회원가입 오류

**수정:**
```xml
<!-- ✅ 이메일 존재 여부는 상태와 무관하게 확인 -->
<select id="existsByEmail" resultType="int">
    SELECT COUNT(*) FROM MEMBER WHERE member_email = #{memberEmail}
</select>
```

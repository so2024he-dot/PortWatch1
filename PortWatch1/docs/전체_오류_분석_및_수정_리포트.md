## Q2. 로그인/회원가입 오류 분석

### 2-1. 현재 로그인 흐름

```
POST /member/login (memberEmail, memberPass)
  → MemberController.login()
    → memberService.login(email, pass)
      → memberMapper.findByEmail(email)  ← DB 쿼리 (HikariCP 연결 필요)
        → BCrypt.matches(pass, DBhash)
          → 성공 시 세션 설정 → dashboard 리다이렉트
```

### 2-2. 로그인 실패 원인

| 원인 | 내용 | 해결 |
|------|------|------|
| DB 연결 실패 | HikariPool 30초 타임아웃 (IPv6 DNS) | setenv.sh IPv4 설정 |
| 계정 없음 | MEMBER 테이블에 테스트 계정 미등록 | 아래 SQL 실행 |
| member_status ≠ ACTIVE | findByEmail에 `AND member_status='ACTIVE'` 조건 있음 | status 업데이트 |
| BCrypt 해시 불일치 | DB에 저장된 해시가 다른 비밀번호 | 해시 재생성 |

## Q9. AWS RDS 연결 Default 값 충돌

→ 별도 파일 참조: `docs/AWS_RDS_Default값_충돌_분석.md`

**핵심 3가지 해결책 요약:**

### ① EC2 setenv.sh (가장 중요)
```bash
sudo bash -c 'echo "#!/bin/bash
export JAVA_OPTS=\"-Djava.net.preferIPv4Stack=true -Xms256m -Xmx512m\"" \
> /opt/tomcat/bin/setenv.sh'
sudo chmod +x /opt/tomcat/bin/setenv.sh
```

### ② root-context.xml 핵심 설정값 (현재 적용됨)
```
connectTimeout=10000     : TCP 연결 10초 (default: 무제한)
connectionTimeout=30000  : 풀 대기 30초 (30000 > 10000 ✅)
initializationFailTimeout=-1 : Tomcat 기동 허용 (default=1ms → 즉시실패)
sslMode=DISABLED         : MySQL 8.4 SSL 비활성 (useSSL=false 만으론 부족)
```

### ③ 올바른 EC2 배포 절차
```bash
sudo systemctl stop tomcat
sudo rm -rf /opt/tomcat/webapps/ROOT
sudo rm -f  /opt/tomcat/webapps/ROOT.war
sudo cp /home/ubuntu/portwatch.war /opt/tomcat/webapps/ROOT.war
sudo systemctl start tomcat
# ← ROOT.war 절대 삭제 금지! (삭제하면 404)
sleep 30
curl -s http://localhost:8080/debug/db-check
```

- `allowPublicKeyRetrieval=true` 미포함 → MySQL 8.4.7 `caching_sha2_password` 인증 실패

**mysql CLI는 성공하는 이유**: CLI는 OS 기본 인증 방식 사용, JDBC는 별도 드라이버 설정 필요.

## 2. DDL 분석 (스크린샷 143905 - PuTTy MySQL 출력)

### 2-1. 발견된 테이블: MEMBERS (레거시)
```sql
SELECT members_id, member_name, member_email, member_point, member_nick, member_status
FROM MEMBERS;
```

| 컬럼명 | 비고 |
|-------|------|
| `members_id` | PK (복수형 's' 포함) |
| `member_name` | 이름 |
| `member_email` | 이메일 |
| `member_point` | 포인트 (현재 스키마에 없음) |
| `member_nick` | 닉네임 (현재 스키마에 없음) |
| `member_status` | 상태 (일부 빈 값) |

### 2-2. MEMBERS vs MEMBER 테이블 비교

| 구분 | MEMBERS (레거시) | MEMBER (현재 코드) |
|------|----------------|-----------------|
| PK | `members_id` | `member_id` |
| 추가 컬럼 | `member_nick`, `member_point` | `member_phone`, `member_address`, `member_gender`, `balance` |
| 사용 여부 | ❌ 미사용 (이전 설계) | ✅ 현재 코드 사용 |

### 2-3. MEMBERS 테이블 데이터 현황
```
admin_nick  → admin@portwatch  (member_status: 빈값)
admin_test  → test@test.com    (member_status: 빈값)
test1       → test@naver.com   (member_status: 빈값)
```

⚠️ **중요**: `member_status`가 빈 문자열 → 우리 코드의 `AND member_status = 'ACTIVE'` 조건에 불일치 → 로그인 실패 가능

### 2-4. 권장 조치

**Option A: MEMBERS 테이블 삭제 (권장)**
```sql
USE portwatch;
-- 데이터 백업 후
DROP TABLE IF EXISTS MEMBERS;
```

**Option B: 데이터를 MEMBER 테이블로 이전**
```sql
USE portwatch;
-- test@test.com이 MEMBER 테이블에 없으면 이전
INSERT IGNORE INTO MEMBER (member_id, member_email, member_name, member_status)
SELECT members_id, member_email, member_name, 'ACTIVE' FROM MEMBERS;
```

## 3. 현재 MEMBER 테이블 스키마 (코드 기준)

```sql
CREATE TABLE IF NOT EXISTS MEMBER (
    member_id      VARCHAR(50)  NOT NULL,
    member_email   VARCHAR(100) NOT NULL UNIQUE,
    member_pass    VARCHAR(200) NOT NULL,
    member_name    VARCHAR(20)  NOT NULL,
    member_phone   VARCHAR(20)  NULL,
    member_address VARCHAR(255) NULL,
    member_gender  VARCHAR(10)  NULL,
    member_birth   TIMESTAMP    NULL,
    member_role    VARCHAR(20)  NOT NULL DEFAULT 'USER',
    member_status  VARCHAR(20)  NOT NULL DEFAULT 'ACTIVE',
    balance        DOUBLE       NOT NULL DEFAULT 1000000,
    created_at     TIMESTAMP    NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at     TIMESTAMP    NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    PRIMARY KEY (member_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

## 4. Mapper 연결 구조 분석

```
MemberServiceImpl
  └── @Autowired MemberMapper (interface: com.portwatch.mapper.MemberMapper)
        └── XML: MemberMapper_mapper_pkg.xml
              namespace="com.portwatch.mapper.MemberMapper"

        └── XML: MemberMapper.xml
              namespace="com.portwatch.persistence.MemberDAO"
              → findEmail, findById, selectMemberById, insert, update, delete ✅
```

→ 이전 WAR에서 `findEmail` 메서드명 사용 → 현재 코드는 `findByEmail` (수정됨)
→ 새 WAR 배포 시 자동 해결

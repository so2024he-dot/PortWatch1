# PortWatch 주식목록 데이터 연결 문제 분석

> 작성일: 2026-03-31 | Spring 5.0.7 + MyBatis 3.5.6 + AWS RDS MySQL 8.4.7

---

## 1. 문제 현상

`http://54.180.142.111:8080/stock/list` 접속 시 **"주식 목록 (0개 기업)"** 으로 표시됨.

## 3. 발견된 문제점 (3가지)

---

### 3-1. [최고 심각도] EC2 배포본 JDBC URL 오타

**원인:** EC2에서 `sed` 명령으로 파일을 수정할 때 `/`를 구분자로 사용 → URL 내의 `/` 때문에 치환이 잘못됨.

**현재 EC2 파일** (`/opt/tomcat/webapps/ROOT/WEB-INF/spring/root-context.xml`):
```
❌ portwatch-db.seoul.ap-northeast-2.rds.amazonaws.com
```

**정상 URL**:
```
✅ portwatch-db-seoul.cpggwqm1goo3.ap-northeast-2.rds.amazonaws.com
```

**영향:** DB 연결 자체가 불가능 → `getAllStocks()` 예외 발생 → null 반환 → 0건 표시

**EC2에서 직접 수정 방법:**
```bash
sudo nano /opt/tomcat/webapps/ROOT/WEB-INF/spring/root-context.xml
```

변경 전:
```
portwatch-db.seoul.ap-northeast-2.rds.amazonaws.com
```

변경 후 (전체 jdbcUrl 속성):
```xml
<property name="jdbcUrl"
    value="jdbc:mysql://portwatch-db-seoul.cpggwqm1goo3.ap-northeast-2.rds.amazonaws.com:3306/portwatch?useSSL=false&amp;sslMode=DISABLED&amp;serverTimezone=Asia/Seoul&amp;characterEncoding=UTF-8&amp;allowPublicKeyRetrieval=true&amp;connectTimeout=30000&amp;socketTimeout=60000"/>
```

저장: `Ctrl+O` → `Enter` → `Ctrl+X`

재시작:
```bash
sudo systemctl restart tomcat
```

확인:
```bash
curl -s http://localhost:8080/debug/db-check
```

---

### 3-2. [심각도 중] stock/list.jsp 변수명 오류 (이미 수정됨)

**원인:** JSP 테이블 섹션에서 `${stockList}` 사용 (컨트롤러는 `${stocks}` 전달)

**수정 전** (stock/list.jsp:486):
```jsp
<c:forEach items="${stockList}" var="stock">
```

**수정 후** (✅ 이미 커밋됨):
```jsp
<c:forEach items="${stocks}" var="stock">
```

---

### 3-3. [심각도 하] StockServiceImpl 예외 시 null 반환

**위치:** `StockServiceImpl.java:53-58`

**현재 코드:**
```java
public List<StockVO> getAllStocks() {
    try {
        return stockMapper.findAll();
    } catch (Exception e) {
        log.error("전체 주식 조회 실패: {}", e.getMessage());
        return null;   // ❌ null 반환 → NullPointerException 위험
    }
}
```

**수정 코드:**
```java
public List<StockVO> getAllStocks() {
    try {
        return stockMapper.findAll();
    } catch (Exception e) {
        log.error("전체 주식 조회 실패: {}", e.getMessage());
        return java.util.Collections.emptyList();  // ✅ 빈 목록 반환
    }
}
```

> **참고:** StockController(line 79)에서 이미 null 체크를 하고 있어 현재는 NPE가 발생하지 않음. 하지만 다른 코드에서 직접 호출 시 위험.

---

### 3-4. [성능 이슈] findAll에 LIMIT 없음

**위치:** `StockMapper.xml:56-59`

**현재 코드:**
```xml
<select id="findAll" resultMap="StockResultMap">
    SELECT <include refid="stockColumns"/> FROM STOCK
    ORDER BY country ASC, stock_code ASC
</select>
```

**문제:** 전체 데이터를 한 번에 조회 → 데이터 증가 시 응답 지연

**수정 코드** (✅ 이미 적용):
```xml
<select id="findAll" resultMap="StockResultMap">
    SELECT <include refid="stockColumns"/> FROM STOCK
    ORDER BY country ASC, stock_code ASC
    LIMIT 200
</select>
```

## 4. MySQL 데이터 검증 쿼리

EC2에서 MySQL 접속 후 실행:

```sql
-- 1. STOCK 테이블 존재 여부
SHOW TABLES LIKE 'STOCK';

-- 2. 데이터 건수
SELECT COUNT(*) FROM STOCK;

-- 3. 국가별 건수
SELECT country, COUNT(*) AS cnt FROM STOCK GROUP BY country;

-- 4. 샘플 데이터
SELECT stock_code, stock_name, country, current_price FROM STOCK LIMIT 5;
```

**STOCK 테이블이 비어있을 경우 → 크롤링 실행:**
```

```

## 5. STOCK 테이블 스키마 (DDL)

```sql
CREATE TABLE IF NOT EXISTS STOCK (
    stock_id     INT AUTO_INCREMENT PRIMARY KEY,
    stock_code   VARCHAR(20)    NOT NULL UNIQUE,
    stock_name   VARCHAR(100)   NOT NULL,
    country      VARCHAR(10)    DEFAULT 'KR',
    market       VARCHAR(20),
    market_type  VARCHAR(20),
    industry     VARCHAR(100),
    current_price DECIMAL(15,2) DEFAULT 0,
    price_change  DECIMAL(15,2) DEFAULT 0,
    change_rate   DECIMAL(8,4)  DEFAULT 0,
    volume        BIGINT        DEFAULT 0,
    market_cap    DECIMAL(20,2) DEFAULT 0,
    updated_at   DATETIME,
    created_at   DATETIME       DEFAULT CURRENT_TIMESTAMP
);
```

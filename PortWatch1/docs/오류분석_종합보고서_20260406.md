## ★ 핵심 결론 (3줄 요약)

> 1. **로컬 STS**: Spring Context는 정상 기동. DB 연결만 실패 → 원인: STS JVM에 `-Djava.net.preferIPv4Stack=true` 미설정으로 IPv6 DNS가 RDS 호스트명을 해석 못 함
> 2. **EC2**: 신규 WAR 배포 완료, Spring 정상 기동, DB 연결 상태는 EC2에서 `curl` 로 직접 확인 필요
> 3. **/news/refresh 405**: `@PostMapping` 전용 엔드포인트 → 브라우저 주소창(GET) 접근 불가, curl -X POST 사용

## 5. 수정 필요 항목 정리

### [즉시] Fix 1: STS Tomcat JVM 아규먼트 추가
**증상**: `Inet6AddressImpl.lookupAllHostAddr` → `UnknownHostException`
**원인**: STS JVM에 `-Djava.net.preferIPv4Stack=true` 없음
**수정**:
```
STS 메뉴: Servers 탭 → Apache Tomcat v9.0 더블클릭
  → "Open launch configuration" 링크 클릭
  → [Arguments] 탭 → "VM arguments:" 박스 맨 끝에 추가:
     -Djava.net.preferIPv4Stack=true
  → [Apply] → [OK]
  → Tomcat 재시작
```

### [즉시] Fix 2: AWS RDS 공개 접근 설정 (로컬 개발용)
**증상**: 로컬 Windows에서 RDS 엔드포인트 연결 불가
**원인**: RDS가 VPC 내 프라이빗 서브넷에만 있거나, 공개 IP 없음
**수정**:
```
AWS Console 로그인 → RDS → 데이터베이스
→ portwatch-db-seoul 클릭
→ [수정(Modify)] 버튼
→ "연결(Connectivity)" 섹션 → "공개적으로 액세스 가능(Publicly accessible)" = 예(Yes)
→ [계속] → "즉시 적용" 선택 → [클러스터 수정]

RDS 보안 그룹 추가:
→ RDS → 보안 그룹 → 인바운드 규칙 편집
→ [규칙 추가]: 유형=MySQL/Aurora, 포트=3306, 소스=내 IP (또는 0.0.0.0/0 개발용)
→ [규칙 저장]
```

### [즉시] Fix 3: EC2 DB 연결 확인
```bash

curl -s http://localhost:8080/debug/db-check | python3 -m json.tool
```
